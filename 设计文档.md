# MultiLED

一个面向嵌入式系统的轻量级 LED 状态管理框架。

设计思想参考 `MultiButton`，通过对象化管理和状态机实现多个 LED 的统一控制。

---

# 1. 项目简介

在嵌入式项目中，LED 控制通常比较简单：

```c
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_1, GPIO_PIN_SET);
```

但是随着项目复杂度增加，会出现：

- 多个 LED 控制代码分散
- 闪烁逻辑重复实现
- 延时阻塞任务
- 状态切换混乱
- FreeRTOS 下需要多个定时器管理

MultiLED 的目标是：

> 使用一个统一的状态机框架管理所有 LED 行为。

支持：

- 常亮
- 常灭
- 慢闪
- 快闪
- 指定次数闪烁
- 心跳灯
- 后续扩展呼吸灯、PWM等效果

# 6. 数据结构

## 6.1 LED模式

```c
typedef enum
{

    LED_MODE_OFF = 0,

    LED_MODE_ON,

    LED_MODE_BLINK_SLOW,

    LED_MODE_BLINK_FAST,

    LED_MODE_BLINK_N_TIMES,

    LED_MODE_HEARTBEAT,


}led_mode_t;
```

---

## 6.2 LED对象

```c
typedef struct led led_t;


struct led
{

    uint8_t id;


    /*
     * 当前模式
     */
    led_mode_t mode;


    /*
     * 当前输出状态
     */
    uint8_t state;


    /*
     * 时间基准
     */
    uint32_t tick;


    /*
     * 闪烁周期
     */
    uint32_t period;


    /*
     * 闪烁次数
     */
    uint16_t blink_count;


    /*
     * 目标次数
     */
    uint16_t blink_target;


    /*
     * GPIO输出函数
     */
    void (*write_pin)
    (
        uint8_t id,
        uint8_t level
    );


    /*
     * 链表
     */
    led_t *next;

};
```

# 9. GPIO抽象

MultiLED 不直接依赖 HAL。

错误：

```c
HAL_GPIO_WritePin();
```

正确：

```c
led->write_pin(
        led->id,
        level
);
```

用户实现：

```c
void led_write_pin
(
    uint8_t id,
    uint8_t level
)
{

    switch(id)
    {

        case LED_RUN:

            HAL_GPIO_WritePin(
                LED_RUN_GPIO_Port,
                LED_RUN_Pin,
                level
            );

            break;


        case LED_FAULT:

            HAL_GPIO_WritePin(
                LED_FAULT_GPIO_Port,
                LED_FAULT_Pin,
                level
            );

            break;

    }

}
```

# 14. 设计原则

MultiLED遵循：

1. 业务代码不直接操作GPIO

2. LED行为由状态机管理

3. 所有动作非阻塞

4. 硬件接口抽象

5. 支持RTOS和裸机

6. 简单API实现复杂效果
